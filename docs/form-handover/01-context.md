# 01. ビジネス背景・LP戦略

## sekai stay の事業

民泊運営代行スタートアップ。代表は劉 添毅。AIネイティブな運営により従来15-20%が標準のレベニューシェアを **8% + 月額1万円/物件** に下げてシェアを獲得する戦略（2026-05-01 確定）。

現在の主力物件:
- LAKE HOUSE 野尻湖
- The Lake Side INN
- MOUNTAIN VILLA ニセコ
- Matsumoto物件

## 運用モード（2026-05-01 確定）

- **基本 = 自社運用モード**: OTA アカウント・Beds24・宿泊契約 すべてセカイチ名義
- **オプション = 広告代理モード**: OTA アカウントはオーナー名義／Payout はセカイチ口座（追加オプション費）

## 物件診断レポート申込フォームの位置づけ

### 目的
オーナー（民泊事業者）が自分の物件Airbnb URLや売上を入力すると、**「現在の代行手数料 vs sekai stay 8% でいくら損してるか」を可視化したレポート** をメール送付。レポート閲覧後にオンラインミーティング予約に繋げる、というリード獲得の一次接点。

### 現状の構成
- **LP本体**: sekaistay.com（明神 洸次郎 管理）
- **フォーム本体**: japanvillas.kss-cloud.com/report-request（吉蔵 = 吉田 管理）← LPに iframe 埋込
- **問題**: フォームを変えたい時、LP管理者と分離してて変更が遅い

### 今回の意思決定（2026-05-04）
**フォームの所有権をテンイチ側（sekaistay.com）に移管**。理由:
1. LPとフォームが同じ人の管理下にある方が改善サイクルが早い
2. 外向き = テンイチ／内向き = 吉蔵 という役割分担を明確化
3. テンイチもAIエージェント（Claude Code 等）を使えるので、コード書き換え自体は可能

### 移管後の役割分担
- **テンイチ**: LP + フォーム + データ保管場所
- **吉蔵**: 受信した顧客データ → 内向きポータル → 営業フォロー・レポート自動生成・メール送信
- **接続**: Webhook（テンイチ→吉蔵）
