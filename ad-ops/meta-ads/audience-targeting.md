# Meta Ads オーディエンス設計

> **目的**: Meta（Facebook + Instagram）で 25 リード/月を獲得するためのオーディエンス構造設計。
>
> **CPA 想定**: ¥6,000〜¥12,000（リターゲ ~¥4,000、類似 ~¥8,000、興味関心 ~¥12,000）
>
> **媒体特性**: 検索意図のない受動的な層に届く。クリエイティブ訴求力 = 90%、オーディエンス精度 = 10%（Meta AI が学習する前提）。

---

## キャンペーン構造

```
Campaign 1: SS-Meta-Retargeting    日予算 ¥2,000  最も CV が安い・必須
Campaign 2: SS-Meta-Lookalike       日予算 ¥3,000  類似オーディエンス・スケール用
Campaign 3: SS-Meta-Interest        日予算 ¥3,000  興味関心・新規発掘
```

合計: ¥8,000/日 → 月¥24万。残り予算は好調な広告セットに増額配分。

---

## Campaign 1: SS-Meta-Retargeting（最優先）

**狙い**: sekaistay.com に既に訪問した人を追いかける。最も CV が安い。

### Meta Pixel カスタムオーディエンス（既存訪問者）

> **前提**: Meta Pixel が30日以上稼働している必要がある。新規ピクセルだと母集団がゼロから蓄積。

| オーディエンス名 | 定義 | 期待規模 |
|---|---|---|
| **SS-Visitors-30d** | 過去30日に sekaistay.com を訪問・全ページ | データ蓄積後 |
| **SS-LP-Visitors-30d** | 過去30日に `/lp` または `/lp/simple` を訪問 | 同上 |
| **SS-Engaged-30d** | 過去30日に sekaistay.com で 25% 以上スクロール | 同上 |
| **SS-FormStart-30d** | 過去30日に「フォーム開始」したが送信せず（離脱層） | 同上・最重要 |

> **初心者ポイント**: ピクセル稼働開始から **最低14日経過** しないとリターゲ規模が小さすぎて配信されにくい。広告未配信時期の Pixel 蓄積が大事。

### 顧客リストカスタムオーディエンス

| オーディエンス名 | ソース | 規模目安 |
|---|---|---|
| **SS-Customers-Excluded** | HubSpot CRM の既存契約済みオーナー（除外用） | テンイチ確認要 |
| **SS-LineFriends-Excluded** | LINE@ 友だち（既にナーチャリング中なので除外） | テンイチ確認要 |

> **エクスクルージョン用**: 既存顧客に「リード獲得広告」を出すのは無駄なので除外。

### 広告セット設計

| 広告セット | 含む | 除外 | 配信先 |
|---|---|---|---|
| Set A | SS-FormStart-30d | SS-Customers | 全プラットフォーム |
| Set B | SS-Engaged-30d | SS-FormStart-30d, SS-Customers | 全プラットフォーム |
| Set C | SS-Visitors-30d | SS-Engaged-30d, SS-FormStart-30d, SS-Customers | Feed のみ |

> Set A → Set B → Set C と熱量降順。予算配分も A:B:C = 4:3:3 が初期推奨。

## Campaign 2: SS-Meta-Lookalike（スケール用）

**狙い**: 既存リード/顧客に類似する新規層を Meta AI に発掘させる。

### 類似オーディエンス（Lookalike Audience / LAL）

| ソース | 類似率 | 推定規模 | 用途 |
|---|---|---|---|
| **HubSpot CRM 契約済みオーナー** | 1% | 約 10万-15万人 | 高精度・最優先 |
| **HubSpot CRM 商談化リード（成約問わず）** | 1% | 同上 | 母集団広め・探索的 |
| **`SS-FormStart-30d`** | 1% | 同上 | リード意欲層に類似 |
| **HubSpot CRM 契約済み** | 3% | 約 30万-45万人 | スケール用・1%が枯れたら |

> **初心者ポイント**: Lookalike のソースは **最低100件、推奨1000件以上**で精度が出る。HubSpot 契約済みが100件未満なら、まず「LP訪問者」「フォーム開始者」をソースにするか、リード母集団が貯まるまで Lookalike は保留。

### 広告セット設計

| 広告セット | オーディエンス | 配信先 |
|---|---|---|
| LAL-1 | 契約済み 1% LAL | Feed + Stories + Reels |
| LAL-2 | フォーム開始 1% LAL | Feed + Stories + Reels |
| LAL-3 | 契約済み 3% LAL | Feed のみ |

## Campaign 3: SS-Meta-Interest（興味関心）

**狙い**: Lookalike と Pixel が育つまでの初期配信。Meta の興味関心ターゲ。

### 興味関心の組み合わせ案

| 広告セット | 興味関心 | 補足条件 |
|---|---|---|
| **Set 1: Airbnb ホスト** | Airbnb / Vrbo / Booking.com / Superhost / 民泊 | 年齢 30-65・日本在住 |
| **Set 2: 不動産投資家** | 不動産投資 / 賃貸経営 / 投資用不動産 / アパート経営 | 年齢 35-65・日本在住・世帯収入上位50% |
| **Set 3: 宿泊事業者** | ホテル経営 / 旅館経営 / 宿泊業 | 年齢 35-65・日本在住 |
| **Set 4: 一般・地理エリア** | （興味関心なし）| 年齢 35-60・観光地周辺市区町村のみ |

> **検証仮説（learnings.md と連動）**:
> - Set 2「不動産投資家」は CPA 高い予測（投資家層は自分で運用したい傾向）
> - Set 1「Airbnb ホスト」が最有望
> - 1週間後に Set 1 を残してそれ以外を削るパターンが多い

## 配信プラットフォーム

| プラットフォーム | 推奨度 | 理由 |
|---|---|---|
| **Facebook Feed** | ⭐⭐⭐⭐⭐ | 30-65歳の主要層に届く・最も CV 取れる |
| **Instagram Feed** | ⭐⭐⭐⭐ | 30-45歳に届く・ビジュアル訴求 |
| **Instagram Reels** | ⭐⭐⭐ | 縦動画素材が必要・素材次第 |
| **Instagram Stories** | ⭐⭐⭐ | 縦動画/画像必要 |
| **Audience Network** | ⭐ | アプリ内広告・誤クリック多い → **OFF 推奨** |
| **Messenger** | ⭐ | CV 効率悪い → OFF 推奨 |

> **初心者ポイント**: Meta はデフォルトで「Advantage+ 配置」が ON で、Audience Network も含まれる。**最初は手動で Feed/Reels/Stories のみに絞る**ほうが CPA が安定する。データが貯まってから Advantage+ に戻す判断。

## オーディエンス重複の管理

| 広告セット同士 | Meta が自動で除外 | 手動除外要否 |
|---|---|---|
| 同一キャンペーン内の異なる広告セット | ❌（重複入札してくる） | ✅ 必須 |
| 異なるキャンペーンの広告セット | ❌（同上） | ✅ 必須 |

具体的に:
- Campaign 1（リターゲ）の対象は Campaign 2/3 から **除外**
- Campaign 2（LAL）の対象は Campaign 3 から **除外**
- Campaign 3（興味関心）はこれ以上重複しないので除外不要

> **初心者ポイント**: 重複を放置すると、自分のキャンペーン同士で入札合戦して CPC が上がる。Meta Audience Insights → 「オーバーラップ」で重複率を確認できる。**20% 以上ならアラート**。

---

## 配信目的

すべてのキャンペーンで **「コンバージョン → Lead」** を目的に設定。

> **初心者ポイント**: 「リーチ」「エンゲージメント」目的にすると、CV しないユーザーにも配信されて CPA が悪化する。最初は「CV最適化」一択。
