# note Content Bank — SEKAI STAY

> **対象**: note 記事（経営者論考軸）
>
> **読者ターゲット**: 地方の物件オーナー（年配層含む）・横文字や略語に馴染みのない読者を想定
>
> **トーン**: AI 言及ゼロ・落ち着いた経営者論考・けんすう note 寄り・横文字最小化（必要なら必ず日本語注記）
>
> **文字数**: 2,500-5,500 字（章立て `##` あり）
>
> **発信頻度**: 月 7 本目標 (第1週 3 本でスタート)
>
> **スキル参照**: `shared/docs/japanese-business-sns-copywriting.md` §13 note 専用ルール

## note 戦略の位置づけ

note は X とは **完全別軸** のチャネル。

| 観点 | X 長文 | note |
|---|---|---|
| 読者文脈 | スクロール中・興味本位 | 検索流入・じっくり読みに来る |
| 目的 | フォロワー獲得・拡散 | 信頼蓄積・SEO 起点 |
| AI 言及 | ✅ 押し出す | ❌ ゼロ |
| トーン | 引用・問いかけ・ぶっちゃけ | 自己開示・構造分析・物語 |
| 字数 | 1,500-2,500 | 2,500-5,500 |
| 締め | 体言止め・余韻 | 静かな問いかけ |

**X 長文をそのまま転載しない**。同じテーマでも書き直す。

## 3 軸構成

### A. 経営者自己開示型
創業時の意思決定・失敗・葛藤を書く。「〜を決めた日」「〜の時に考えていたこと」。
AI/効率化は黒子で、表に出さない。

### B. 業界構造分析型
業界のコスト構造・価格構造・労働構造を分解する論考。
データ・比較表 OK。経営学的フレーム（PPM・5Forces・LTV/CAC 等）を引いても良い。

### C. オーナー成功物語型
1 物件・1 オーナーの再生物語。時間軸・登場人物・転換点を持つ長編。
ADR や稼働率の数字は出すが、AI が触ってる話はしない。

## 初週 3 本 要約

### N-1: 民泊代行を 8% でやろうと決めた日 (A 自己開示型・約 1,800 字)

**章立て**:
1. 「相場」を疑うところから始まった
2. コストの中身を、ゼロから見直す
3. 「1 人で 100 物件」を成り立たせる仕組み
4. 8% でも、品質を上げられる確信があった
5. これからのこと

**フック**: 「民泊代行という事業を始めるとき、業界の平均的な手数料は 15-25% でした。それを 8% でやると決めた日のことを、書いておこうと思います。」

**狙い**: 創業者ストーリー軸・スーパーホスト運営力 + 仕組み化判断の経営者論考化

---

### N-2: 民泊代行の中身を、お金の流れから見直す (B 構造分析型・約 2,200 字)

**章立て**:
1. 民泊代行のお金の流れ
2. 「人件費」が手数料を縛っている
3. やり方を変えると、何が起きるか
4. 8% は、やり方が変われば成立する
5. 競合と何が違うか
6. これからの民泊代行業

**フック**: 「民泊代行という業態は、なぜ手数料が下がらないのか。業界の中にいる人間として、整理しておきたいと思います。」

**狙い**: SEO 上位狙い（「民泊代行 手数料」「民泊代行 コスト」検索）+ 業界分析者ポジション獲得・「P/L」「経営学的」など固い表現を回避

---

### N-3: サウナを置いて見せ方を変えたら、平均宿泊単価が 2 倍になった話 (C 物語型・約 2,300 字)

**章立て**:
1. 「稼働率は高いのに、稼げない」物件
2. 「立地が中途半端」という見立て
3. お客さんの層を変える
4. 1 泊 ¥26,000 → ¥58,000 になった
5. なぜこの変化が起きたか
6. これからのこと

**フック**: 「ある京都の 1 棟貸し物件で、平均宿泊単価（1 泊あたりの平均価格）が 2 倍になった話を書きます。派手な改装をしたわけではありません。やったのは、物件の見せ方を変えただけです。」

**狙い**: スーパーホスト運営力をストーリーで実感させる・/switch/founder への自然導線・「再ポジショニング」「ADR」「リスティング」を全て日本語化

## ネタストック (Backlog)

### A 自己開示型
- スーパーホストになるまでの 5 年間で、捨てた選択肢の話
- 京都で築 30 年の町家を初めて借りた日のこと
- 値下げで稼働率を取りに行って失敗した時期
- オーナーから「もう代行は信じない」と言われた時の話
- 起業 1 年目、Beds24 を選んだ理由
- 民泊新法施行（2018）の時、現場で起きていたこと

### B 構造分析型
- ダイナミックプライシングは、なぜ 80% の物件で機能していないか
- 一棟貸しと区分マンション、収益構造はどう違うか
- インバウンド回復で「勝てる物件」と「勝てない物件」の分かれ目
- OTA 手数料の構造（Airbnb / Booking / 楽天トラベル）
- 民泊代行業界の M&A はなぜ起きないか
- 民泊新法 vs 旅館業法 vs 特区民泊、コスト構造の比較

### C 物語型
- 値下げ競争に巻き込まれた京都町家を、5 ヶ月で再生した話
- インバウンド枯渇期に、国内法人客にピボットしたオーナーの話
- ペット可で再ポジショニングしたら、稼働率が 95% になった物件
- 「家具を全部 IKEA に統一したら、修繕費が 1/3 になった」物件
- 清掃クレーム連発の物件を、運用切替で 4.9 評価まで戻した話
