## 1. エグゼクティブサマリー

- **長文投稿（280-4000字）× 個人2アカウント二刀流 + 架空社員サブ**の3アカウント体制で、X のロング投稿優遇アルゴリズムに乗せる

## 5. 長文投稿 × 広告連動設計（Boost the Winners モデル）

### 5-1. アルゴリズム前提

X（2025-2026）のアルゴリズムは:
- **滞在時間（Dwell Time）が最重要シグナル** — 長文Bは滞在時間を稼ぐので推薦面で優遇
- **リプライ・引用RT率がエンゲージスコアに直結** — 議論を呼ぶコピーが伸びる
- **外部リンクは表示順を下げる** — 長文末尾のLPリンクは「2投稿目」or プロフィール誘導で代替

### 5-2. Boost the Winners 運用フロー

```
[Day 0] テンイチ / ジロー / 架空社員が長文B 投稿
   ↓ 24-48h オーガニック単体で配信
[Day 2] X Ads Manager で impression / engagement / link click を測定
   ↓ エンゲージ率（≥3%）+ link click 数（≥10）の上位 20% を抽出
[Day 3] 該当投稿を "Promoted Post" として X Ads に投入
   - 既存のオーガニック投稿そのまま広告化（広告感ゼロ）
   - 入札: Webサイトクリック単価 ¥150-300（手動 CPC）
   - ターゲ: 既存 X Ads 9パターンと同じオーディエンス
[Day 7] Promoted Post の CPL / CTR / CVR を測定
   - CPL ¥15K 以下なら継続
   - CPL ¥20K 超で 3 日継続なら停止
   - 勝者は次週も予算追加（最大 ¥3,000/日）
```

### 5-3. 既存 X Ads 9パターンとの並列運用

| 配信種別 | 役割 | 月予算 |
|---|---|---|
| **既存 X Ads 9パターン（price/portal/trust × 3）** | コールド配信・新規オーディエンス開拓 | ¥80,000（¥2,667/日） |
| **Boost the Winners（オーガニック投稿の広告化）** | 温度感のあるリーチ・広告感ゼロ | ¥70,000（¥2,333/日） |
| **合計（X 月予算）** | — | **¥150,000** |

→ 既存予算枠（¥150K/月）内で完結。増額提案なし。

### 5-4. CTA 設計（長文末尾）

長文B の末尾は以下のテンプレ:


URL は `https://sekaistay.com/switch?utm_source=x&utm_medium=organic&utm_campaign=longform&utm_content=<account>_<topic>` で計測。Promoted 化した投稿は `utm_medium=cpc&utm_campaign=boost-the-winners` に書き換えてリポスト or 別投稿として配信。

## 6. 計測体制

### 6-1. 計測スタック

```
ユーザー
  ↓
X 上で長文B 投稿に遭遇（オーガニック or Promoted）
  ↓
末尾 URL クリック → LP /switch* （GA4 + X Pixel + Meta Pixel 設置済み）
  ↓
フォーム送信 → POST /api/report-requests/submit
  ↓ 3 系統並列転送
  ├─ Supabase lead_submissions（utm_source=x で識別）
  ├─ 吉蔵 CRM（forwardLead）
  └─ Meta CAPI（sendMetaCapiLead・hashed PII 付き）
  ↓
X Ads CV「Website Conversion」（X Pixel + URL ベース）
```

### 6-2. 計測 ID 一覧

- **X Pixel**: ROADMAP Phase 3 で取得予定（env: `NEXT_PUBLIC_X_PIXEL_ID`）
- **GA4**: `G-B7M920RCGR`（既存・全媒体共通）
- **Meta Pixel / CAPI**: `1658477098524563` / `989839370242915`（既存）

### 6-3. CV 設計

| CV 名 | ソース | 用途 |
|---|---|---|
| **X Lead**（X Pixel + URL ベース） | フォーム送信（thanks ページ到達） | X Ads CV 最適化対象 |
| **utm_source=x 内訳**（Supabase） | フォーム送信の utm パラメータ | オーガニック vs Promoted の内訳分析 |

### 6-4. UTM 設計

```
オーガニック投稿:
?utm_source=x&utm_medium=organic&utm_campaign=longform&utm_content=tenichi_<topic>

Promoted（Boost the Winners）:
?utm_source=x&utm_medium=cpc&utm_campaign=boost-the-winners&utm_content=<original_post_id>

既存 X Ads 9パターン:
?utm_source=x&utm_medium=cpc&utm_campaign=ss-cold-202605&utm_content=<creative_id>
&utm_term=price|portal|trust
```

`lp_variant` は既存の `<PageViewTracker>` 経由で GA4 / Meta Pixel / Supabase に伝播。

## 7. リスク管理（X 特有）

### 7-1. プラットフォームリスク

| リスク | 影響 | 対処 |
|---|---|---|
| **架空社員アカウントの BAN** | コンテンツ資産消失 | bio で「運営チームメンバー」明記・実在性を担保 |
| **長文投稿のアルゴリズム変動** | リーチ激減 | 月次でアルゴリズム動向観測・3アカウント分散でリスク分散 |
| **広告ポリシー違反**（比較表現等） | 配信停止 | 「業界相場と比べて」等の緩和表現を統一 |
| **薬機法・特商法** | 措置命令リスク | 「保証収入」「絶対」等 NG ワード辞書を Pre-publish チェックに組込 |

### 7-2. オペレーションリスク

| リスク | 影響 | 対処 |
|---|---|---|
| **ジロー X 運用工数超過**（Unplugged + Xだと負荷高） | 投稿頻度低下 | ジロー本人の合意必須・週2本長文Bが上限・Claude下書き活用 |
| **テンイチ・ジロー間の発信内容重複** | フォロワーの飽き | 5本柱×3アカウント切り口マトリクス（4-2）で構造的に防止 |
| **Promoted 化判断の遅延** | 24-48h ウィンドウ逃し | 月-金 毎朝 09:00 に判定タスク（Inbox 提案） |

### 7-3. 透明性・倫理

- **架空社員ペルソナ**: 「SEKAI STAY 運営チームメンバー視点をペルソナ化」と公式に説明可能な状態にする。固有名「@ss_unei_chan」のような bio に「中の人は複数いる場合あり」明記推奨
- **ステマ規制**: テンイチ・ジロー本人発信は自然な意見表明としてOK。Promoted 化した投稿には X の標準「広告」表示が自動付与されるので問題なし

| Promoted CPL | ¥10K-15K | ¥7K-12K | ¥5K-10K |

### 8-2. なぜ M1 でリードが少ないか

1. **フォロワー基盤がゼロから** — 個人アカウントの長文投稿は被フォロー数に比例した拡散をする
2. **アルゴリズム学習期** — 各アカウントの長文投稿は最初の 5-10 本まで impressions が伸びない傾向
3. **5/22 プロダクトローンチ + PR TIMES 配信のレバレッジを M1 後半から享受** — ローンチ当日の砲火で 5 月末から 6 月にかけてフォロワー急増想定（当初 6/1 想定から前倒し）

### 8-3. 目標達成パス

| 月 | 主な施策 | 期待効果 |
|---|---|---|
| M1（6月） | PR TIMES 同時砲火・長文B 週5本のリズム確立 | フォロワー基盤 1,500・初期 Promoted データ蓄積 |
| M2（7月） | Boost the Winners の勝者パターン特定・予算集中 | CPL 30% 改善・リード倍増 |
| M3（8月） | 長文B コンテンツバンクを 50 本まで蓄積 | リード 15-20 件で X チャネル黒字化（広告 ROAS > 1） |
