# X Ads オーディエンス設計

> **目的**: X (旧 Twitter) で 15 リード/月を獲得するためのオーディエンス構造設計。
>
> **CPA 想定**: ¥8,000〜¥15,000（民泊系は他媒体より高い傾向）
>
> **媒体特性**: テキスト主体・リアルタイム性が高い。**キーワードターゲティング**が最強（過去のツイート/エンゲージから関心を抽出）。Meta のような「興味関心」AI 推定より、X は「実際に発言した言葉」で抽出するためニッチBtoB と相性が良い。

---

## キャンペーン構造

```
Campaign 1: SS-X-Keyword          日予算 ¥2,000   X 特有・最優先
Campaign 2: SS-X-Follower-LAL     日予算 ¥1,500   著名アカウントのフォロワー類似
Campaign 3: SS-X-Retargeting      日予算 ¥1,500   X Pixel 蓄積後に投入
```

合計初期: ¥3,000/日（慣らし期1週間）→ Campaign 1 単独で開始。
2週目以降: ¥5,000/日 で勝者キャンペーンへ集中。

> **X 運用ポリシー**: X はアルゴリズム学習が Meta より弱く、3 キャンペーン並列より「**キーワード単独で勝てるパターンを見つけてから拡張**」が早い。

## Campaign 1: SS-X-Keyword（最優先・X 特有）

**狙い**: 「民泊代行に困っている」「物件運営に悩んでいる」と過去にツイート/エンゲージした層に直接届ける。X 特有の最強ターゲティング。

### キーワード設計（過去30日のツイート/エンゲージで判定）

#### 訴求パターン別キーワード

| パターン | 含むキーワード | 訴求 LP |
|---|---|---|
| **価格主導** | 民泊代行 / Airbnb代行 / 手数料 / 高い / 割合 / 業者 | `/switch` |
| **ポータル主導** | Airbnbホスト / 物件管理 / 複数物件 / 稼働率 / ダイナミックプライシング / OTA | `/switch/portal` |
| **信頼主導** | 民泊 始め方 / Airbnb 投資 / 不動産投資 / 副業 / スーパーホスト | `/switch/founder` |

#### 除外キーワード

> 競合・無関係領域を排除する。

- 「副業詐欺」「在宅ワーク」「投資詐欺」（投資詐欺関連の誤マッチ防止）
- 「ホテル予約」「Airbnb 予約」「民泊 泊まる」（ゲスト側）
- 競合代行名（後でテンイチに確認・大手3社の名前は除外推奨）

### マッチタイプ

| マッチ | 用途 | 注意 |
|---|---|---|
| **Phrase Match (フレーズ一致)** | 推奨デフォルト | 「民泊 代行」を含むツイート全部にマッチ |
| **Exact Match (完全一致)** | ニッチな鉄板キーワードのみ | リーチ激減・テスト後 |
| **Broad Match (部分一致)** | 推奨せず | 無関係なツイートにマッチ・ムダ消費 |

> **初心者ポイント**: X のキーワードは「ツイート本文」だけでなく「リプライ・引用 RT・いいね」も対象になる。母集団はそこそこ大きい。

### 広告セット設計

| 広告セット | キーワード群 | 訴求パターン |
|---|---|---|
| KW-Price | 民泊代行 / Airbnb代行 / 手数料 / 高い / 業者 | 価格主導 → `/switch` |
| KW-Portal | Airbnbホスト / 物件管理 / 複数物件 / 稼働率 / OTA | ポータル主導 → `/switch/portal` |
| KW-Trust | 民泊 始め方 / Airbnb 投資 / 不動産投資 / 副業 | 信頼主導 → `/switch/founder` |

初週は KW-Price のみ ¥3K/日で配信 → CTR・CPA で勝負筋を判断 → 2週目に勝者へ ¥5K/日集中。

## Campaign 2: SS-X-Follower-LAL（フォロワーターゲティング）

**狙い**: 民泊・Airbnb・不動産投資関連の著名アカウントのフォロワー（および類似ユーザー）に届ける。

### 対象アカウント候補（フォロワーが見ている層）

> テンイチ確認要：以下のアカウントが日本市場で適切か・追加候補があるか

| カテゴリ | アカウント候補 | 補足 |
|---|---|---|
| 公式 | `@Airbnb_jp` `@Booking_jp` | 民泊オーナー / ゲスト混在・絞り込み弱い |
| 民泊専門メディア | （調査要） | 「民泊新法」「Airbnbマガジン」等 |
| 不動産投資インフルエンサー | （テンイチ推奨者を聞く） | アカウント名要確認 |
| 業界キーパーソン | 楽待・健美家・REINS 関連 | 不動産投資家層 |
| 競合代行 | （要確認） | 競合のフォロワー = 既存代行ユーザー = 価格パターンの母集団 |

### 設計方針

| 広告セット | ターゲット | 訴求 |
|---|---|---|
| FOL-1 | 競合代行アカウントのフォロワー類似 | 価格主導（乗り換え訴求） |
| FOL-2 | Airbnb 公式 + 民泊メディアのフォロワー類似 | ポータル主導（運用改善訴求） |
| FOL-3 | 不動産投資インフルエンサーのフォロワー類似 | 信頼主導（参入支援訴求） |

> **重要**: フォロワーターゲティングはアカウントを5-10件まとめてグループ化する方が安定する（単一アカウントだとリーチが小さく学習しない）。

## Campaign 3: SS-X-Retargeting（X Pixel 蓄積後・第2フェーズ）

**狙い**: sekaistay.com 訪問者を X 上で追いかける。

### 前提

- X Pixel 設置後、**最低14日経過 + 1,000+ オーディエンス蓄積** が必要
- 開始時期: **2026-06 前半以降** が現実的

### カスタムオーディエンス案

| オーディエンス | 定義 | 用途 |
|---|---|---|
| **SS-X-Visitors-30d** | 過去30日 sekaistay.com 訪問者 | リターゲ全般 |
| **SS-X-LP-Visitors-30d** | 過去30日 `/switch` 系訪問者 | ホットリード |
| **SS-X-FormStart-30d** | フォーム開始したが未送信 | 最重要・離脱層 |

### 顧客リスト除外

- 既存契約済みオーナー（CRMエクスポート）
- LINE@ 友だち（オプション）

## 興味関心ターゲティング（補助）

X の興味関心は Meta より粒度が荒い。Campaign 1/2 と組み合わせて絞り込む補助役割。

| 興味関心 | 関連カテゴリ |
|---|---|
| Business & finance > Real estate | 不動産・投資家層 |
| Travel > Hotels & accommodations | 民泊・旅行業 |
| Business > Entrepreneurship | 副業・起業層 |

> 単独で使うとリーチ過剰になり CPA 悪化。**キーワード ∩ 興味関心** の AND 条件で絞ると効果的。

## デモグラフィック設定（全 Campaign 共通）

| 項目 | 設定値 | 理由 |
|---|---|---|
| 国 | 日本 | 当然 |
| 言語 | 日本語 | 当然 |
| 年齢 | **35-65 歳** | 民泊オーナー層は中年〜シニア中心。20代は除外でムダ消費削減 |
| 性別 | 全性別 | 民泊オーナーは性別差少ない |
| デバイス | iOS + Android のみ（PC OFF） | X モバイル中心・PC は誤クリック多い |
| 配信時間 | 全時間（後で最適化） | 初期は学習データ確保 |

## 配信フォーマット

| フォーマット | 推奨度 | 理由 |
|---|---|---|
| **プロモーション広告（ツイート広告）** | ⭐⭐⭐⭐⭐ | 標準・280文字 + 画像/動画 |
| **Website Card** | ⭐⭐⭐⭐⭐ | LP誘導必須・自動 OG 取得 |
| **動画広告** | ⭐⭐⭐⭐ | 15-30秒の本人語り動画があれば信頼パターンで強い |
| **カルーセル** | ⭐⭐⭐ | 3 訴求を1広告に詰めるテスト用・優先度低 |
| **フォロワー獲得広告** | OFF | 今回はCVが目的なので不要 |

> **必須**: X は画像/動画なしで広告配信不可。**最低1点の画像が必須**。

## オーディエンス重複の管理

| 重複ケース | 対処 |
|---|---|
| Campaign 1（KW）と Campaign 3（リターゲ）の重複 | リターゲは KW から除外する |
| Campaign 1（KW）内の異なる広告セット | キーワード上は重複しない設計（Price/Portal/Trust で分離済み） |
| Campaign 2（Follower）と Campaign 1（KW）の重複 | データ蓄積後に確認・初期は許容 |
